# 仮想売買バックテスト仕様

## 目的

既存のイベントスタディを変更せず、売買代金モメンタムイベントを実際の売買ルールとして扱った場合の成績を比較する。

既存の `backtest` は、イベント日から21/63/126/252営業日後のリターンを確認するイベントスタディとして残す。
仮想売買バックテストは、別コマンド `trade-backtest` として実行する。

## 共通前提

* イベント検出条件は既存の売買代金モメンタム条件を使用する
* 対象銘柄ユニバースも既存バックテストと同じ条件を使用する
* エントリー日はイベント日の翌営業日とする
* エントリー価格は翌営業日の始値とする
* 翌営業日の始値が存在しない場合、そのイベントは売買対象外とする
* 売買コスト、税金、スリッページは初期実装では考慮しない
* 同一イベントを複数戦略で評価し、戦略間で比較できるようにする

## 比較する戦略

### Strategy A

現在のイベント条件に対して、翌営業日始値で仮想エントリーする。
その後、設定された固定保有営業日数まで保有し、最終日の終値で決済する。

初期値:

```text
固定保有日数 = 21営業日
```

### Strategy B

Strategy A と同じイベントを翌営業日始値で仮想エントリーする。
ただし、ATRベースの損切と利確を設定する。

初期値:

```text
ATR期間 = 14営業日
損切幅 = ATR14 × 1.5
損切価格 = エントリー価格 - 損切幅
利確価格 = エントリー価格 + 損切幅 × 2.0
最大保有日数 = 21営業日
```

保有中の判定:

* 安値が損切価格以下になった場合、損切価格で決済
* 高値が利確価格以上になった場合、利確価格で決済
* 同じ日に損切価格と利確価格の両方に到達した場合、保守的に損切を優先する
* どちらにも到達しない場合、最大保有日の終値で決済する

### Strategy C

Strategy B に相対20日リターンフィルタを追加する。

初期値:

```text
相対20日リターン = 銘柄の20営業日リターン - TOPIX代替ベンチマークの20営業日リターン
採用条件 = 相対20日リターン >= 0
```

条件を満たさないイベントは、Strategy C では売買対象外とする。

## 出力

仮想売買バックテストは、CSVと日本語Markdownの両方を出力する。

出力先:

```text
reports/trade_backtest/
```

出力ファイル:

* `trades.csv`
* `summary.csv`
* `strategy_comparison.csv`
* `report.md`

## 集計指標

各戦略について以下を集計する。

* トレード数
* 勝率
* 平均損益率
* 中央値損益率
* 平均利益
* 平均損失
* 期待値
* 利確率
* 損切率
* 期限決済率
* 平均保有営業日数
* 最大ドローダウン

## 注意事項

本機能は売買代金モメンタムイベントを売買ルールとして扱った場合の検証であり、売買推奨ではない。
初期実装では売買コスト、税金、スリッページを考慮しないため、実運用成績とは差が出る。
